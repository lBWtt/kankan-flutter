# 任务⑭：发动态 compose 屏改「朋友圈」样式

**先读** `docs/KANKAN_SPEC.md`。目标：把 `lib/features/publish/compose_screen.dart` 的排版改成**微信朋友圈发布器**那种干净、留白、九宫格图片的样子。只重排版式，**不改发送逻辑、不改 Post 模型、不动我刚修的 `_close()` 返回逻辑**。

> ⚠️ 你看不到设计。朋友圈发布器的关键特征：整屏白底大留白；顶部文字输入区最显眼；图片是**九宫格缩略图 + 末尾"+"添加格**；发表按钮是右上绿色胶囊（空内容置灰）。

## 版式（从上到下）
1. **顶栏**：左「取消」（纯文字，`t2`）+ 右「发表」绿色胶囊（`KkColors.teal`，空内容 `t4` 置灰不可点）。中间可留空或极淡标题。**保留现有 `_close()` 取消逻辑，别改**。
2. **文字输入区**：大号多行 `TextField`（`KkType.body`，无边框，placeholder「这一刻的想法…」或沿用现有 hint），占据显眼位置、上下留白充足。字数计数 `0/500` 若保留就做淡（`t3`，右下角小字），别抢眼。
3. **九宫格图片区**（signature）：`_media` 用 **3 列网格**（`GridView` 或 `Wrap`，每格近正方形、圆角 `KkRadius.sm`、间距 `KkSpacing.xs`）：
   - 每张图缩略图 + 右上角小「×」移除。
   - 末尾一个**虚线"+"添加格**（同格尺寸，`Icons.add`，`t3`），点击加图（复用现有 `MediaPicker` 的选图回调 / 逻辑，别引新依赖）。
   - 满 9 张时隐藏"+"格（`Post` 最多 9 图）。
4. **引用项目卡**：保留（放图片下方），可移除（现有逻辑）。
5. **已选话题**：选中的话题显示为一排 `#tag` chip（可删），放引用项目下方或输入区下方。
6. **底部操作条**：只放**已实现**的入口——加图（若已在九宫格"+"里就不重复）/ 话题 / 引用项目。**不要加**「所在位置」「提醒谁看」「谁可以看」这类朋友圈原生但本 App 没实现的项（零假按钮铁律）。@ 提及若当前没真功能也别放假的。
